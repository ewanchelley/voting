How To Use This Website

In an election, there are multiple candidates. For example an election may have 3 candidates: Adam, Beth and Chris.

Each voter produces a ranking, which is their order of preference for the list of candidates. For example, if a voter believed Adam was the best candidate and Beth was the worst, their ranking would be Adam, Chris, Beth. Once all votes have been cast, there is a list of rankings. For example, suppose 10 voters ranked the candidates introduced above. The list of rankings produced may look like:

V1: Adam, Chris, Beth
V2: Chris, Beth, Adam
V3: Beth, Adam, Chris
V4: Adam, Chris, Beth
V5: Adam, Chris, Beth
V6: Beth, Adam, Chris
V7: Chris, Adam, Beth
V8: Adam, Chris, Beth
V9: Chris, Adam, Beth
V10: Beth, Chris, Adam

So who won the election?

The challenge now is to determine the order of the candidates in the final standings, which should reflect the preferences of the voters as a whole. This may seem trivial at first, but there are many different algorithms used to accomplish this, each having advantages and disadvantages. This application allows the user to input their own set of rankings and candidates to compare how different preference aggregation algorithms work.

Begin by editing the candidates and list of rankings by clicking the Edit Rankings button on the sidebar. Rankings are input as a comma separated list containing each of the candidates names. The candidates and rankings can also be imported from a txt file or generated randomly.

Once the candidates and rankings have been input, select an algorithm from the sidebar to run. Algorithms are run as soon as the page is loaded, using the rankings that the user has input.
